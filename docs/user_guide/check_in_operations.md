# 签到操作

## 手动签到

### 单个账号

1. 在账号页面找到账号卡片
2. 点击**签到**按钮
3. 等待操作完成
4. 查看结果提示通知

### 签到过程

1. **会话检查**：验证缓存的会话是否仍然有效
2. **WAF 绕过**（如需要）：启动浏览器绕过 Cloudflare
3. **API 请求**：向服务商发送签到请求
4. **余额更新**：获取并缓存新余额
5. **历史记录**：将签到结果保存到数据库

## 批量签到

一次签到多个账号：

1. 在账号页面，使用复选框选择账号
2. 点击**批量签到**按钮
3. 在对话框中监控进度
4. 完成后查看结果

**提示**：
- 禁用的账号不包含在批量操作中
- 失败的签到不会停止批量操作 - 所有账号都会尝试
- 结果显示每个账号的成功/失败状态

## 自动签到

### 启用自动签到

1. 编辑账号
2. 将**自动签到**切换为启用
3. 设置签到时间（小时和分钟）
4. 保存账号

### 自动签到工作原理

- NeuraDock 必须正在运行才能执行自动签到
- 每天在计划时间执行签到
- 如果应用在计划时间关闭，签到将被跳过
- 结果记录到签到历史

### 推荐设置

| 设置 | 建议 |
|-----|------|
| 时间 | 清晨（6:00-9:00）以避开高峰时段 |
| 分散 | 不同账号设置不同时间以避免速率限制 |

## 签到状态

| 状态 | 含义 |
|-----|------|
| 成功 | 签到成功完成 |
| 失败 | 签到失败（查看错误消息） |
| 进行中 | 签到正在运行 |
| 已计划 | 自动签到已安排 |

## 常见签到错误

| 错误 | 原因 | 解决方案 |
|-----|------|---------|
| Session expired | Cookies 无效 | 更新 cookies |
| WAF bypass failed | 浏览器自动化失败 | 检查浏览器安装 |
| Network error | 连接问题 | 检查网络 |
| Already checked in | 今天重复签到 | 等到明天 |
| Rate limited | 请求过多 | 等待后重试 |

## 签到历史

查看过去的签到记录：

1. 导航到仪表盘
2. 在活动流中查看最近的签到
3. （即将推出）详细历史页面

**注意**：签到历史功能尚未完全实现。
