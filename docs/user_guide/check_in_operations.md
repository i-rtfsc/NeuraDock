# 签到操作

## 手动签到

### 单个账号

1. 在账号页面找到账号卡片
2. 点击**签到**按钮
3. 等待操作完成
4. 查看结果提示通知

### 签到过程

1. **会话检查**：验证缓存的会话是否仍然有效
2. **代理配置**（如启用）：通过配置的代理服务器发送请求
3. **WAF 绕过**（如需要）：启动浏览器绕过 Cloudflare
4. **API 请求**：向服务商发送签到请求
5. **余额更新**：获取并缓存新余额
6. **历史记录**：将签到结果保存到数据库

### v0.5.0 新功能：代理配置

在"偏好设置"页面可以配置应用代理：
- **代理类型**：支持 HTTP、HTTPS 和 SOCKS5
- **代理地址**：格式为 `host:port`，例如 `127.0.0.1:7890`
- **应用范围**：所有签到、余额查询和 Token 获取请求都将通过代理

## 批量签到

一次签到多个账号：

1. 在账号页面，使用复选框选择账号
2. 点击**批量签到**按钮
3. 在对话框中监控进度
4. 完成后查看结果

**提示**：
- 禁用的账号不包含在批量操作中
- 失败的签到不会停止批量操作 - 所有账号都会尝试
- 结果显示每个账号的成功/失败状态

## 自动签到

### 启用自动签到

1. 编辑账号
2. 将**自动签到**切换为启用
3. 设置签到时间（小时和分钟）
4. 配置签到间隔（可选，默认无限制）
5. 保存账号

### v0.5.0 新功能：可配置签到间隔

现在可以为每个账号设置自定义签到间隔，避免频繁签到导致的限制：

- **默认间隔**：0小时（无限制，仅按自动签到时间执行）
- **自定义间隔**：可设置为 0-24 小时之间（0 表示无时间限制）
- **用途**：适应不同服务商的签到要求，例如设置 24 小时确保每天只签到一次

### 自动签到工作原理

- NeuraDock 必须正在运行才能执行自动签到
- 根据配置的间隔执行签到（而非固定每天）
- 如果应用在计划时间关闭，签到将被跳过
- 结果记录到签到历史，可在签到记录页面查看

### 推荐设置

| 设置 | 建议 |
|-----|------|
| 时间 | 清晨（6:00-9:00）以避开高峰时段 |
| 间隔 | 设置 24 小时以确保每天只签到一次，避免重复签到 |
| 分散 | 不同账号设置不同时间以避免速率限制 |

## 签到状态

| 状态 | 含义 |
|-----|------|
| 成功 | 签到成功完成 |
| 失败 | 签到失败（查看错误消息） |
| 进行中 | 签到正在运行 |
| 已计划 | 自动签到已安排 |

## 常见签到错误

| 错误 | 原因 | 解决方案 |
|-----|------|---------|
| Session expired | Cookies 无效 | 更新 cookies |
| WAF bypass failed | 浏览器自动化失败 | 检查浏览器安装 |
| Network error | 连接问题 | 检查网络 |
| Already checked in | 今天重复签到 | 等到明天 |
| Rate limited | 请求过多 | 等待后重试 |

## 签到历史

查看过去的签到记录：

1. 导航到**签到记录**页面
2. 选择要查看的账号
3. 查看连续签到统计、日历视图和趋势分析
4. 点击日历中的日期查看详细记录

### v0.5.0 签到分析功能

签到记录页面现在提供高级分析：

- **月度概览**：查看本月签到情况、收益统计
- **趋势图**：30天签到和收益趋势可视化
- **日历深度链接**：点击日期跳转到详细记录
- **断签提醒**：自动识别并标记缺失的签到
