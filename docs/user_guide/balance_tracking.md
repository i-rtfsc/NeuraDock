# 余额追踪

## 查看余额

余额信息显示在每个账号卡片上：

- **配额**：总分配配额（GB）
- **已用**：本周期已使用量
- **剩余**：可用配额

## 余额缓存

为减少 API 请求，NeuraDock 缓存余额数据：

| 设置 | 默认值 | 描述 |
|-----|-------|------|
| 缓存时间 | 1 小时 | 余额被视为过期的时间 |

### 缓存工作原理

1. 当你查看账号时，如果余额过期则获取
2. 签到后，余额自动刷新
3. 手动刷新无论缓存状态都获取新数据

### 配置缓存时间

1. 转到**设置**页面
2. 找到**余额缓存时间**设置
3. 调整小时数（1-24）
4. 更改立即生效

## 手动刷新

强制刷新账号余额：

1. 找到账号卡片
2. 点击刷新图标
3. 等待余额更新

## 余额历史

NeuraDock 跟踪余额随时间的变化：

- 每天最多一条记录
- 存储在数据库中
- （即将推出）余额历史图表

## 理解余额值

| 值 | 含义 |
|---|------|
| 配额 | 计费周期内分配的总 GB |
| 已用 | 目前已消耗的 GB |
| 剩余 | 配额 - 已用 |
| 最后检查 | 上次获取余额的时间 |
| 新鲜 | 数据是否在缓存时间内 |

## 余额指示器

| 指示器 | 含义 |
|-------|------|
| 绿色 | 剩余 > 50% |
| 黄色 | 剩余 20-50% |
| 红色 | 剩余 < 20% |
| 灰色 | 余额未知 |

## 余额问题故障排除

**余额显示"--"或"未知"**：
- 账号从未检查过
- API 请求失败
- 点击刷新获取余额

**余额似乎不正确**：
- 缓存可能过期 - 点击刷新
- 服务商 API 可能有延迟
- 直接在服务商网站检查

**签到后余额不更新**：
- 检查签到是否成功
- 某些服务商延迟更新余额
- 几分钟后手动刷新
